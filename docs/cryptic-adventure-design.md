# Cryptic Adventure Design

Status: design note

## Thesis

There is a puzzle-design discipline that is not cryptography, not cryptic crosswords, and not general riddle-writing.

Call it cryptic adventure design:

```text
designing problems that are cryptic enough to feel discovered,
but legible enough that any given three people can find some solution
in about ten minutes
```

The object is not secrecy. The object is calibrated discovery.

## What It Is Not

This is not actual cryptography.

Actual cryptography protects information against adversaries. It assumes hostile readers, hidden keys, mathematical hardness, and security under attack.

Adventure puzzles should not do that. The players are not attackers. They are guests in a room that wants, eventually, to open.

This is also not cryptic-crossword design.

Cryptic crosswords use formal clue conventions, wordplay, abbreviations, containers, reversals, homophones, and setter culture. That is a beautiful machine, but it is not this machine.

Adventure puzzles need table motion, shared noticing, object handling, feedback, and partial progress. They live in a room, not only in a clue.

## The Target

A good cryptic adventure puzzle should be:

- initially strange
- materially inspectable
- fair after the fact
- solvable by several routes or partial insights
- discussable by a small group
- recoverable after a wrong first theory
- satisfying when the hidden rule clicks

It should not require:

- private author mind-reading
- obscure real-world facts
- specialist puzzle-culture knowledge
- a single exact phrase
- total silence until the final answer
- one player becoming the designated solver while everyone else watches

## The Ten-Minute Table Rule

Design for three ordinary attentive players and ten minutes.

That does not mean every group solves it in ten minutes. It means the puzzle should produce enough traction that a group can make real progress without guessing what genre of cleverness the author wanted.

Useful signs:

- one player can notice a surface detail
- another can suggest a move
- another can connect feedback to the rule
- wrong moves still teach something
- the final step uses the discovered rule once, cleanly

Bad signs:

- the puzzle has no feedback until the end
- the solution depends on a pun
- the clue vocabulary is doing private symbolic work
- the group can generate five equally plausible theories
- the puzzle gets harder after every interaction without explaining why

## Cryptic Enough

Cryptic enough means the answer is not immediately visible from the surface.

Good sources of cryptic pressure:

- a hidden invariant
- a rule attached to position instead of object
- a phase shift after observation
- an information-encoding trick
- a symmetry break
- a cost for premature revelation
- a witness statement that kills one possible world
- a resource bottleneck that is not the visible resource

The puzzle should make the group say:

```text
wait, it is doing that
```

not:

```text
how were we supposed to know that?
```

## Not Too Cryptic

Not too cryptic means the group has handles.

Give them:

- visible state
- small object count
- clear actions
- immediate or delayed-but-predictable feedback
- at least one redundant confirmation
- language that names the relevant domain
- enough examples to infer the rule without a lecture

A puzzle can hide its rule. It should not hide its interface.

## Fairness Is Not Easiness

Fair does not mean obvious.

Fair means that after the solution, the group can look back and see that the puzzle was asking for that kind of thought.

Unfair:

```text
The silver key opens the door because silver means mercy in the author's private lore.
```

Fair:

```text
Every merciful object is marked with the same silver notch, and using two notched objects reveals the pattern before the lock requires it.
```

The first asks for lore. The second teaches a rule.

## Group Solving

Adventure puzzles are social objects.

They should support:

- parallel inspection
- theory swapping
- small experiments
- partial credit
- visible progress
- a graceful hint path

If only one person can work on the puzzle at a time, the puzzle should be very short. If it takes ten minutes, it needs shared surface area.

## Goblet Relevance

Goblet should not generate cryptic adventure puzzles from vibes.

Goblet can help audit the structured parts:

- how many worlds exist before clues
- how many worlds each clue kills
- whether multiple solutions survive
- whether a clue discriminates no worlds
- whether the intended solution requires an unstated rule
- whether interaction is required
- whether feedback makes the hidden rule learnable

This is the useful overlap:

```text
not cryptography
not crossword cluecraft
small-world inference with calibrated opacity
```

## Design Questions

For any proposed adventure puzzle, ask:

1. What is the hidden rule?
2. What can players touch, test, ask, or compare?
3. What does the first wrong move teach?
4. How many plausible worlds survive each clue?
5. What assumption is the author silently relying on?
6. Can three people discuss it productively?
7. Is there a partial solution if they miss the intended one?
8. Does the final action use the discovered rule, or merely name the answer?

## Short Definition

Cryptic adventure design is the craft of making small playable mysteries whose rules are hidden but learnable.

The door should not open because the players guessed the author's thought.

The door should open because the players learned how the door thinks.
